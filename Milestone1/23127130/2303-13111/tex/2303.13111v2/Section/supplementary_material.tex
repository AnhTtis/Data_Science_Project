\subsection*{Resolution Sensitivity}
\textbf{Definition} \textit{A network cannot deal with a flexible resolution input once the number of parameters is set.}~\cite{liu2022mlpsurvey} \\
\textbf{Vanilla MLP} Given input feature maps $\mathbf{X}\in \mathbb{R}^{H\times W \times D \times C}$, the number of parameters could be set only with prior knowledge of $H$, $W$, $D$ and $C$. \\
\textbf{Axial decomposition} Given an input slice of feature maps $\mathbf{X}\in \mathbb{R}^{H\times W \times C}$, the shape of $X$ requires $C = kW$ which means width has to be a divisor of the number of channel once the number of parameter is fixed. \\
\textbf{MLPP} Given an input slice of feature maps $\mathbf{X}\in \mathbb{R}^{H\times W \times C}$, the shape of $X$ requires ${\rm cd}(C,W) = L$, which only requires segment length $L$ to be a common divisor of the number of channel and width, thus accepting a more flexible input shape. Additionally, by setting $g = C/L$, the number of parameters of our method does not depend on the shape of input feature maps.
